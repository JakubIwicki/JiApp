How should we handle the Android app icon?

Currently: default React Native green robot icon. Your logo is rectangular — app icons must be square.

Adaptive Icon (Modern)
Circle
Squircle
Square

Android 8+ — system clips to device shape.
Foreground image + solid background layer.

Pros

  • Matches modern Android look
  • System handles shape variations
  • Supports animated/parallax effects

Cons

  • Requires vector drawable or precise PNG
  • Two layers to design (fg + bg)
  • More setup files
Legacy PNG Icons (Classic)
Square Icon

5 density buckets (mdpi → xxxhdpi).
Simple square PNG, no layering.

Pros

  • Simpler — just resize PNG
  • Works on all Android versions
  • One image, no layers to manage

Cons

  • Looks outdated on modern Android
  • No adaptive shape support
  • Stands out awkwardly among adaptive icons
A

Adaptive Icon (recommended)

Modern approach. Create foreground + background layers. Use a simplified/extracted square portion of the logo as the foreground.

B

Legacy PNG Icons

Generate square PNG at 5 densities. Simpler but less polished on modern Android.

C

Both (full coverage)

Adaptive icon for Android 8+ with legacy PNG fallback for older devices. Best of both worlds.
